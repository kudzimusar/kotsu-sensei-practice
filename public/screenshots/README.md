# Screenshots Directory

This directory contains app screenshots for PWA store listings (Google Play, Microsoft Store, etc.).

## Required Screenshots

### Mobile (form_factor: "narrow")
Recommended size: 540x720 or 1080x1920 pixels

1. **home-mobile.png** - Home screen showing main features and quiz options
2. **quiz-mobile.png** - Interactive quiz interface with question and answers
3. **planner-mobile.png** - Study planner and schedule view

### Desktop (form_factor: "wide")
Recommended size: 1280x720 or 1920x1080 pixels

1. **home-desktop.png** - Desktop home view with full layout
2. **quiz-desktop.png** - Desktop quiz experience

## Screenshot Guidelines

### Requirements
- **Minimum**: 3-8 screenshots
- **Aspect ratio**: Between 1:2 and 2.3:1
- **Format**: PNG or JPEG
- **Quality**: High resolution, clear and crisp

### Best Practices
1. Show actual app content, not mockups
2. Highlight key features in each screenshot
3. Use real data that looks realistic
4. Include captions/labels in the manifest
5. Show different screens and features
6. Maintain consistent branding

### Content to Capture
- Main dashboard/home screen
- Core functionality (quiz, planner, tests)
- Unique features (road signs, schedules)
- User progress/achievements
- Settings or profile screens

## Taking Screenshots

### For Mobile
1. Open app in mobile device or browser (DevTools mobile view)
2. Set viewport to 540x720 or 1080x1920
3. Capture key screens
4. Optimize file size (keep under 8MB each)

### For Desktop
1. Set browser window to 1280x720 or 1920x1080
2. Capture full app interface
3. Show responsive desktop layout
4. Optimize file size

## Tools
- Browser DevTools screenshot tool
- [Screely](https://screely.com/) - Add device frames
- [AppMockUp](https://app-mockup.com/) - Professional mockups
- Chrome Extension: "Full Page Screen Capture"
